HTML

  • The head element or metadata contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • ul(unorderlist) ol(orderedlist) li(list items).
  • p represents a paragraph or block of text.
  • img is for an image, you need the src for source within the img tag.
  • br stands for a break in content.
  • a stands for anchor

CSS

  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • (https://htmlcolorcodes.com/) a link to the HTML color codes.

Git

  • git status: checks what branch we are currently on.
  • git add -A: Stages changes.
  • git checkout -b branch-name: creates a new branch and switches to it.
  • git commit -m "your message here”: Commits changes from the branch with message.
  • git pull origin main: Pulls from main branch to ensure it is up to date.
  • git push origin Branch/Name: Makes a PR request to the repo for the current branch.

JavaScript

  • A variable is a named container that allows us to store data in our code.
  • Control flow is the order in which a computer executes code in a script.
  • You can use conditional statements and loops to control the order in which code is executed, or if it is even executed at all.
  • The strict equality operator (===) checks to see if two values are equal.
  • An array is a single variable that is used to hold a group of data.
  • Within an array data's location is identified with a unique number called an index starting at 0. Ex(0,1,2,3).
  • Commenting out code makes it inactive without deleting it. Ex (/* comment */).
  • A function is simply a set of instructions that tells the computer how to perform a certain task.
